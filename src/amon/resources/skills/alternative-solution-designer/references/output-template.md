# Output template

預設輸出請依照下面順序，除非使用者要求不同格式。

## 一、問題本質重構

- 用一句話指出：
  - 真正難的不是什麼，而是什麼。
- 再補一句：
  - 為什麼目前解法在解表面問題，不是根因。

## 二、抽象化與結構分類

- 指出 1-2 個結構模型。
- 每個模型都要說：
  - 為何符合
  - 典型解法輪廓

## 三、類比轉移與他域對照

- 至少 2 個他域同構場景。
- 每個場景都要回答：
  - 該場景如何解
  - 為何能類比
  - 如何搬來用

## 四、隱含假設與限制鬆綁

- 列出目前解法的前提條件。
- 分兩欄整理：
  - 可放鬆的限制
  - 鬆綁後會出現的解法可能性

## 五、問題拆解

- 用模組角度拆解，例如：
  - 輸入
  - 處理
  - 判斷
  - 比對
  - 輸出
  - 驗證
  - 人工介入
- 逐段說明：
  - 哪段可被重排
  - 哪段可替換技術
  - 哪段可簡化或移除

## 六、新技術引入

- 列出成熟技術、工具或非技術手段。
- 每項要標註：
  - 可解哪段
  - 是否能讓某段流程消失
  - 成熟度：可馬上用 / 需評估

## 七、替代解法生成

- 至少 3 條，且必須來自不同思維類型。
- 每條方案固定回答：
  - 核心概念
  - 實際作法
  - 為何更簡單或更穩
  - 代價與風險

建議優先覆蓋這些類型中的 3 種以上：
- UI / UX
- 流程重排
- 技術替換
- 半自動 / 人工輔助
- 規則取代模型

## 八、最低摩擦解

- 提供一個幾乎不動系統，只改介面或流程即可改善問題的方案。
- 必須說明：
  - 改哪裡
  - 為何低摩擦
  - 如何驗證是否有效

## 九、建議下一步

- 若有明顯優先路線，直接給建議。
- 補一個最小試驗或 pilot：
  - 做什麼
  - 驗證什麼
  - 失敗了代表什麼
