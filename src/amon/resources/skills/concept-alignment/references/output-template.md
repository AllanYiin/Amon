# Output template

Use this output contract exactly unless the user explicitly overrides it.

## Required heading order

```md
## Concept Alignment

### [關鍵概念]定義
[定義、範圍、單位、幣別、相近概念差異]

### 收集背景知識
[先列知識清單，再寫帶來源註記的細顆粒度背景知識]

### 重大影響的具體事件
[列近期具體事件，附日期、影響與來源]
```

## Section guidance

### `### [關鍵概念]定義`
- `[關鍵概念]` 替換成最能代表任務的主題名。
- 若有多個核心術語，可用粗體術語逐一解釋。
- 數字一定要補單位；金額一定要補幣別；法規一定要補法域。

### `### 收集背景知識`
- 先列出為解題需要的知識清單，再展開細節。
- 若內容涉及事件，補齊人事時地物。
- 若內容涉及研究，補論文來源與 abstract 級摘要。
- 若內容涉及法規，補法條或章節。
- 若內容涉及資料集或統計，補資料來源、期間與定義。
- 若某一類資訊不相關，可以省略，但不要硬湊。

### `### 重大影響的具體事件`
- 每則事件都要寫出具體日期。
- 事件必須是具體可驗證的事件，不是抽象口號。
- 說清楚事件如何改變對主題的理解或後續決策。

## Source annotation pattern

- 重要事實後方附上簡短 annotation，例如：
  - `[來源：機構名，文件/頁面名，日期]`
  - `[來源：論文名，期刊/會議，年份]`
- 若使用 Markdown 連結，保持簡潔，但仍需標機構或文件名。
- 圖表下方額外補一行：
  - `來源註記：來源 A；來源 B`

## Mermaid embedding rule

- 只有在圖能顯著提升理解時才插入。
- 圖要放在最相關的段落，不要全部堆在文末。
- 圖後面必須緊接著來源註記。

## Forbidden behaviors

- 不要使用 canvas。
- 不要在概念對齊後直接開始做方案、規格、程式或簡報。
- 不要只給空泛摘要卻沒有來源或具體脈絡。
