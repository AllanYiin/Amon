# Source Triage

搜尋做得好，不只在於關鍵字，也在於你知道哪些結果值得打開。

## 優先順序

1. 官方或原始來源
- 政府機關、標準組織、專案官方文件、公司 IR、原始資料提供者

2. 直接文件
- PDF、法規全文、年報、規格、白皮書、原始數據頁

3. 高可信二手來源
- 主流媒體、研究機構、知名技術文件整理

4. 一般二手整理
- 部落格、論壇、SEO 內容站

若 1 或 2 已經存在，原則上不要用 4 來支撐核心結論。

## 打分維度

每個候選結果至少快速看四件事：
- Authority：這個網站是不是原始或高權威來源
- Freshness：日期是否符合任務要求
- Relevance：標題與摘要是否真的對應研究問題
- Extractability：打開後能不能直接取得需要的資訊

## 何時應該降級或排除

- 內容只是複製其他網站，沒有新增資訊
- 標題看似相關，但摘要已顯示明顯錯題
- 網域可疑、廣告過多、沒有作者或日期
- 搜尋結果其實指向聚合頁，而不是實際文件

## 去重原則

- 同一 URL 只保留一份
- 同站鏡像或列印版保留較完整、較正式的版本
- 同一新聞事件若多篇內容高度重複，只保留最接近原始來源者

## 何時停止搜尋

可停止的條件：
- 已找到足夠回答問題的原始來源
- 第二輪搜尋沒有新增有價值證據
- 使用者要的是決策輸入，不是無止盡收集

不可停止的條件：
- 只找到二手轉述，還沒找到原始來源
- 任務明確要求最新，但目前結果沒有日期或日期過舊
- 來源彼此衝突，尚未交叉驗證
